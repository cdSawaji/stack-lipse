Deploying to a Remote Swordfish Runtime Server

Ensure that you have defined a new server as described in Creating a New Remote Swordfish Server Definition and that you have converted the plug-in feature project that is to be exported to faceted form.

To deploy a plug-in feature project to a remote Swordfish Server:

  1. In the Servers view, right-click on the Swordfish Runtime server.

    The context menu opens.

  2. Click Add and Remove
  3. In the Add and Remove window, use the Add and Remove buttons to move your plug-in projects from Available list to the Configured list box.
  4. Click Finish to start publishing the feature to the remote server. When the publishing process is successfully completed a message is displayed. An error message is also displayed if the publishing fails, for example this may occur if the connection to the server is refused or dropped.
